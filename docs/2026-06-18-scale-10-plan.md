# Nitro Circuit Overdrive — Scale 1 → Scale 10

## Context

**Nitro Circuit Overdrive** is a 2.5D top-down arcade racer (TypeScript + Vite + Three.js) that's grown into a complete-but-rough vertical slice. The user judges it "scale 1" and wants "scale 10". Concrete problems driving this plan:

- **Severe race-start hitch** ("really bad in the first few seconds") — investigation found `buildMountainPass()` creates **600–1,100 individual meshes** on tunnel tracks, ~140 decor props each **deep-clone geometry AND material synchronously**, plus `renderer.compile()` shader compilation (30–150 ms) and a `receiveShadow` traversal of 800+ meshes — all on the frame that starts the race. Total mesh count ~950–1,350, draw calls ~400–800, no instancing anywhere.
- **Graphics below target** — materials are mostly `MeshLambert`; no SSAO/contact shadows, no vignette/grain/DoF, a flat canvas-gradient "sky" (not a dome), no flat-shading, no rim light.
- **Animation thin** — no wheel spin, no suspension squash/pitch, no countdown/lap pops, no boost trail, no menu/idle motion, no podium.
- **Content thin** — 6 cars, 6 tracks, 1 cup, only Race/Tournament modes.

**Decisions (confirmed with user):** Visual/feel bar = **art of rally** (flat-shaded low-poly, gradient sky, atmospheric fog, dust, photo mode) with **Super Cars** gameplay DNA. Stay on **Three.js** (engines later). Execution order: **performance first**. Content: **full roadmap** (Time Trial + ghosts + leaderboards, Elimination, difficulty, more cars/tracks, liveries).

**Outcome:** a smooth-starting, art-of-rally-grade-looking, content-rich arcade racer, shipped phase-by-phase (each phase independently deployable and visually verifiable).

## PHASE 1 — Kill the race-start hitch & cut draw calls  *(do first)*

Goal: no perceptible start hitch; steady-state draw calls ~400–800 → ~120–200; tunnel tracks no longer spike.

- **1a. InstancedMesh foliage/rocks (biggest win).** In `models.ts` add `getModelMergedGeometry(name)` (cache in a module `Map` like the existing `cache` at `models.ts:7`) returning baked geometry + **shared** material per sub-mesh. In `addDecor()` (`track.ts:~400-522`) bucket the ~140 placements by `(modelName, subMesh)` and emit one `THREE.InstancedMesh` per bucket (set per-instance matrices from the existing pos/rot/scale at `track.ts:452-454`). Stop cloning materials for foliage (same-type trees share one). Keep `rawClone`/`cloneScaled` (`models.ts:64,87`) for **cars/debris** (need per-instance tint at `models.ts:147-156`).
- **1b. Cut mountain pass ~10×.** Rewrite `buildMountainPass()` (`track.ts:273-345`): walls → one extruded ribbon strip per side (reuse the `buildRibbon`/`buildKerb` vertex-strip pattern `track.ts:191-242`) instead of per-segment boxes; rocks → one `InstancedMesh` per rock material; arches → `mergeGeometries` per mouth. ~1,000 meshes → ~8.
- **1c. Merge static decor/flags** (`track.ts:506-521`) into the instanced/merged buckets.
- **1d. Async build behind the loading screen** (`main.ts:260-309`): after painting "LOADING…" (`main.ts:263`) `await` a double-`requestAnimationFrame` before `buildTrack`, so the browser paints before the (now-smaller) block. Use `renderer.compileAsync(scene, camera)` (await while LOADING is up) to hide shader compile (`main.ts:298`). Fire `ensureModelsLoaded()` **at `Game` construction** (`main.ts:78-118`) so the 44 GLBs fetch during the menu, not on first race (`main.ts:265`).
- **1e. Cache the built track** (deterministic for `(def, seed)`): module `Map`. **RISK:** `Race.dispose()` (`race.ts:1183-1191`) disposes everything incl. the track group → must skip cached track (tag `userData.cached`, guard traversal). And `BuiltTrack.obstacles` is mutated at runtime (tanker wreck `race.ts:441`, bridge pillars `track.ts:155`) → clone the obstacles array per race. Test a replay (2nd race same track) explicitly.

Verify: `nitro-local`, start a **tunnel track** (Forest Run/Glacier Gate/Neon Boulevard/Canyon Run), `?perf` shows draw calls in low hundreds and small `buildTrack` ms; headless `autopilot` lap with no console errors; replay is instant.

## PHASE 2 — Graphics to the art-of-rally bar

Goal: still frame reads as art of rally — gradient sky dome, exponential fog, flat-shaded forms, contact AO, vignette + subtle grain, rim light, per-theme grade, static day/night.

- **2a. Sky dome** — replace `makeSkyGradient` flat background (`race.ts:154-217`) with an inward-facing `SphereGeometry(2000)` `BackSide` + small `ShaderMaterial` vertical gradient; colours from `theme.fog` + new `theme.skyTop`.
- **2b. `FogExp2`** in place of `THREE.Fog` (`race.ts:217`), density per theme; match dome horizon to fog colour.
- **2c. Flat shading** — `flatShading:true` on procedural materials (`track.ts` kerbs/rocks/mountain, `carmesh.ts`) and GLB foliage (in the 1a merge helper). Prefer `MeshStandardMaterial({flatShading:true})` so they pick up the existing env map (`race.ts:282`).
- **2d. Post FX stack** in `setupComposer` (`main.ts:133-150`), order: `RenderPass → GTAOPass (contact AO, half-res) → UnrealBloomPass (keep subtle) → VignetteShader → HueSaturationShader (per-theme grade) → FilmPass (subtle grain) → SMAAPass → OutputPass`. Gate GTAO/SMAA behind a **quality setting** (Phase 5).
- **2e. Rim light** — second cool-tint `DirectionalLight` opposite the sun (`race.ts:225`), `castShadow=false`, for edge separation.
- **2f. Per-theme palettes / day-night** — extend `TrackTheme` (`data.ts:85-147`) with `skyTop`, `fogDensity`, `grade{hue,saturation}`, `timeOfDay`; drive sun/ambient/hemi/fog/dome from it. Make NIGHT visibly nocturnal (proof of day/night). Full dynamic time-of-day = stretch.

Verify: `/__shot` each of the 4 themes at the countdown (camera framed at `main.ts:296-297`); compare to art-of-rally reference; watch `?perf` with full post stack on a tunnel track.

## PHASE 4 — Content volume (full roadmap)

- **4a. Procedural track generation** — seed → varied closed control-point loops (long straight + hairpin + mixed corners, non-square) fed into the existing `CatmullRomCurve3` in `buildTrack` (`track.ts:77`); only the `points` source changes. **RISK:** AI line (`race.ts:565-630`) + lap/wrong-way detection (`race.ts:805-833`) assume convex-ish loops → **validate each generated track with headless `autopilot` laps; reject tracks where any racer fails to progress.**
- **4b. More cars** — append to `CARS` (`data.ts:24`) + map GLB in `PLAYER_CAR_MODELS` (`models.ts:10`); use unused `taxi/van/police/delivery` GLBs, drop more Kenney GLBs into `public/models/` + `ALL_MODELS` (`models.ts:30`). Aim 10–12 distinct silhouettes.
- **4c. Time Trial + ghost + local leaderboard** — add `'timetrial'` `GameState` (`main.ts:21`); single racer, `weapons:false` + events off (`race.ts:205,533`). Record player `pos/heading` per ~2 frames → store best ghost per `(carId,trackId)` in `save.ts` (extend `Profile` `save.ts:19-30`); replay as translucent non-colliding mesh (exclude from `resolveCarCollisions` `race.ts:837`). Extend `bestTimes` (`save.ts:29`) to a top-N leaderboard + a new screen mirroring `showTournament` (`screens.ts:182`).
- **4d. Elimination mode** — `'elimination'` state; at each lap cross (`race.ts:806-823`) cull last place via `sortedRacers` (`race.ts:1079`), fade its mesh.
- **4e. Difficulty** — `Settings.difficulty` (`save.ts:12-17`); scale `ramp`/`skill`/`rubber` (`main.ts:239-247`, `race.ts:573-574`) and AI `latGrip` (`race.ts:594`).
- **4f. Verify: headless `autopilot` laps on every new/generated track (no soft-lock); manual Time Trial (ghost + leaderboard write), Elimination (cull per lap), each difficulty (AI pace differs); `/__shot` new tracks/cars/liveries.

## Top risks

1. **Track caching vs `dispose()`** (1e) — reusing a cached track group while `dispose()` frees it → blank track on replay. Guard with `userData.cached`; test replay explicitly. *Highest risk.*
2. **`obstacles` runtime mutation** (1e) — clone the array per race or tanker/pillars accumulate across races.
3. **Procedural tracks vs AI/lap logic** (4a) — validate every generated track headlessly before shipping.
4. **Post-FX cost on low-end** (2d) — 5 full-screen passes; quality-gate GTAO/SMAA, default mid preset.
5. **Extra WebGL contexts** (3d/3e) — keep turntable/podium to one or two low-res renderers.
